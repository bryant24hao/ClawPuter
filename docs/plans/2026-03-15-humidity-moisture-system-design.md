# 湿润度系统设计文档

## 概述

为小龙虾宠物新增「湿润度」养成指标，基于真实天气湿度数据驱动，与现有天气系统、配饰系统、互动系统深度整合，形成完整的"天气 → 湿润度 → 行为 → 互动"闭环。

## 设计理念

小龙虾是水生动物，湿度对它有天然意义。这不是一个独立的小游戏，而是现有天气系统的自然延伸：

- **天气影响环境**（已有）→ 天气特效、配饰
- **湿度影响宠物**（新增）→ 湿润度指标、行为变化、玩家互动

玩家需要"照顾"小龙虾的湿润度，创造类似拓麻歌子的"惦记感"。

## 数据来源

湿润度基于 Open-Meteo API 返回的 `relative_humidity_2m`（当前已接入），每 15 分钟更新。不需要额外硬件传感器。

## 一、湿润度指标

### 1.1 数值模型

| 属性 | 值 |
|------|------|
| 范围 | 0 ~ 4（整数，4 格制） |
| 初始值 | 4（满） |
| 存储 | 内存变量，不持久化到 NVS |

### 1.2 屏幕显示

**位置**：屏幕右上角（与左上角 `[TAB] chat` 提示对称）

**样式**：4 个像素水滴图标，横排排列

- `💧💧💧💧` — 满水（4 格）
- `💧💧💧○` — 正常（3 格）
- `💧💧○○` — 有点干（2 格）
- `💧○○○` — 很干（1 格）
- `○○○○` — 脱水（0 格）

每个水滴约 5×7 像素，间隔 2px，总宽度约 26px。用浅蓝色（filled）和暗灰色（empty）区分。

### 1.3 自然衰减

湿润度随时间自然减少，真实湿度影响衰减速度：

| 真实湿度 | 衰减间隔 | 说明 |
|---------|---------|------|
| > 70% | 40 分钟/格 | 高湿环境，自带补水 |
| 40% ~ 70% | 20 分钟/格 | 正常速度 |
| < 40% | 10 分钟/格 | 干燥环境，水分流失快 |

### 1.4 天气联动（自动回复）

特定天气类型会暂停衰减甚至自动回复湿润度：

| 天气类型 | 湿润度效果 |
|---------|-----------|
| 雨 / 小雨 / 雷暴 | 暂停衰减 + 每 15 分钟自动 +1 格 |
| 雪 | 暂停衰减（雪是固态水，不直接补水） |
| 雾 | 衰减速度减半 |
| 晴 / 多云 / 阴天 | 按湿度正常衰减 |

## 二、玩家互动：H 键喷水

### 2.1 触发

- **按键**：H 键（伴侣模式下，非聊天模式）
- **效果**：湿润度 +1 格（上限 4）
- **冷却**：3 秒，冷却期间按 H 无效
- **条件**：湿润度已满时按 H 无效果，不播放动画

### 2.2 动画 + 音效

按 H 后：
1. 屏幕上显示 3~5 个蓝色像素水花粒子，从小龙虾头顶向上扩散，持续约 0.5 秒
2. 播放短促音效（高频水滴音，类似 `tone(2000, 30)` + `tone(1500, 30)`）
3. 小龙虾播放开心跳跃动画（复用现有 `triggerHappy()`）

### 2.3 右上角提示

喷水成功时，对应的水滴图标从空变满，加一个短暂的闪烁效果（亮 → 暗 → 正常，约 0.3 秒）。

## 三、行为变化

湿润度影响小龙虾的日常行为，等级越低表现越明显：

### 3.1 分级行为表

| 湿润度 | 移动速度 | 自发动画 | 特殊表现 |
|--------|---------|---------|---------|
| 4（满） | 正常 ×1.2 | 偶尔自发跳跃（每 30 秒概率触发） | 走路特别轻快 |
| 3（正常） | 正常 | 无特殊 | 正常状态 |
| 2（有点干） | 正常 ×0.8 | 偶尔停下擦汗（新微动画） | 头顶偶尔冒小汗滴 |
| 1（很干） | 正常 ×0.5 | 几乎不自发移动 | 头顶冒 `...` 气泡 |
| 0（脱水） | 不移动 | 趴下动画（新状态） | 头顶冒 `💧` 求助气泡 |

### 3.2 开心跳跃联动

现有空格/回车键触发的开心跳跃行为，受湿润度影响：

| 湿润度 | 按空格效果 |
|--------|-----------|
| ≥ 2 | 正常跳跃 |
| 1 | 小幅度跳跃（跳得矮一些） |
| 0 | 不跳，原地挣扎动画（暗示"太干了没力气"） |

### 3.3 脱水状态细节

当湿润度降到 0 格时：
- 小龙虾进入「脱水趴下」状态（类似睡觉姿势但眼睛睁着）
- 头顶循环显示 `💧` 水滴求助气泡
- 此状态下按方向键无法移动
- 按 H 喷水后恢复到 1 格，小龙虾站起来，可以正常移动

## 四、配饰系统整合

### 4.1 当前配饰逻辑

```
天气类型 → 配饰
晴/多云 → 墨镜
雨/小雨/雷暴 → 雨伞
雪 → 雪帽
雾/阴天 → 口罩
```

### 4.2 新增：湿润度覆盖规则

湿润度低时，配饰优先显示水壶，覆盖天气配饰：

| 湿润度 | 配饰逻辑 |
|--------|---------|
| ≥ 2 | 显示天气配饰（与现有逻辑一致） |
| ≤ 1 | 显示小水壶（新配饰），覆盖天气配饰 |

**设计意图**：小龙虾太渴了，什么都顾不上了，先找水。给玩家一个维护湿润度的动力——想看到小龙虾戴墨镜就得先保证它不渴。

### 4.3 小水壶配饰设计

- 位置：小龙虾右手/右钳处（与身体相邻）
- 尺寸：约 6×8 像素
- 颜色：浅蓝色壶身 + 白色壶嘴
- 朝向：跟随小龙虾面朝方向翻转

---

## 五、天气模拟模式兼容

现有 Fn+W 天气模拟模式下，湿润度系统的行为：

- 模拟天气时，湿润度的天气联动规则跟随模拟天气类型（比如模拟下雨，湿润度暂停衰减）
- 模拟结束后恢复真实天气的衰减速度
- 喷水交互在模拟模式下正常可用

## 六、桌面宠物同步

### 6.1 状态广播

在现有 UDP 状态广播包中新增 `h`（moisture level）和 `rh`（real humidity %）字段：

```json
{"s":0,"f":0,"m":"companion","x":0.5,"y":0.5,"d":1,"w":2,"t":7.0,"h":3,"rh":65}
```

- `h`：湿润度等级（0~4）
- `rh`：真实环境湿度百分比（0~100）

### 6.2 桌面端状态显示

桌面端有三层信息密度，从兜底到详细：

#### 层级 1：菜单栏（兜底，两个模式通用）

在现有 `ESP32: Connected` 菜单项旁显示湿润度和湿度数值：

```
ESP32: Connected · 💧3/4 · 65%
```

- 始终可见，无论 Follow 还是 Scene 模式，点击状态栏图标就能看到
- 作为兜底信息源，用户随时可查

#### 层级 2：Scene Mode 状态栏（详细）

Scene Mode 是 360×200 的独立窗口，有完整天气场景。在底部状态栏时钟旁融合显示：

```
┌──────────────────────────────────────┐
│              天空 / 天气              │
│                                      │
│            🦞 小龙虾                  │
│▓▓▓▓▓▓▓▓▓▓▓▓ 地面 ▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓▓│
│     21:56 | 7° | 65% 💧💧💧○        │
└──────────────────────────────────────┘
```

- 在现有时钟文本后追加：`| 65% 💧💧💧○`
- 水滴用自绘小圆形，filled = 浅蓝，empty = 暗灰
- 与时钟、温度保持同一行，整体居中
- 湿润度 ≤ 1 格时水滴区域缓慢闪烁（每秒闪一次）

#### 层级 3：Follow Mode 行为暗示（被动感知）

Follow Mode 是透明 sprite 跟随光标，追求轻盈不打扰，**不显示任何常驻 UI**。

仅在湿润度 ≤ 1 格时，小龙虾头顶冒水滴求助气泡（与睡觉冒 Zzz 相同的视觉语言）：

```
   💧
    🦞
```

- 水滴气泡循环浮动（上下轻微漂浮，与 Zzz 动画节奏一致）
- 湿润度 = 0 时气泡更大且闪烁
- 湿润度 ≥ 2 时气泡消失，回到纯净 sprite

### 6.3 桌面端行为同步

- 桌面宠物根据 `h` 值显示对应的行为状态（移动速度、配饰）
- 脱水状态（h=0）时桌面宠物同步趴下
- 喷水互动限定在硬件端键盘，桌面端只负责观察

## 七、实现范围与约束

### 7.1 内存预算

| 项目 | 预估占用 |
|------|---------|
| 湿润度变量 + 计时器 | ~20 bytes |
| 水滴图标绘制代码 | ~200 bytes Flash |
| 水花粒子动画 | ~300 bytes Flash |
| 小水壶配饰 sprite | ~96 bytes PROGMEM (6×8 RGB565) |
| 趴下动画帧 | 复用现有 sleep 帧，无额外开销 |
| **合计** | < 1KB，可忽略 |

### 7.2 不做的事情

- **不做** 湿润度持久化（NVS）——每次开机满值，符合"新一天新开始"的宠物逻辑
- **不做** 成就系统——当前 scope 只做湿润度本身，成就作为未来可选扩展
- **不做** 多个养成指标（饥饿、清洁等）——只做湿润度一个，保持简洁
- **不做** 桌面端喷水交互——互动限定在硬件端键盘

### 7.3 涉及文件预估

| 文件 | 改动 |
|------|------|
| `companion.h` | 新增湿润度字段、水花粒子结构、相关方法声明 |
| `companion.cpp` | 湿润度衰减/回复逻辑、水滴 UI 绘制、水花动画、行为变化、配饰覆盖、喷水响应 |
| `sprites.h` | 新增小水壶 sprite 数据 |
| `main.cpp` | H 键事件处理、状态广播新增 `h` 字段 |
| `state_broadcast.cpp` | 广播包加 `h` 字段 |
| `desktop/.../AppDelegate.swift` | 解析 `h`、`rh` 字段，传递给 PetBehavior 和 PetWindow |
| `desktop/.../PetBehavior.swift` | 根据湿润度调整桌面宠物行为 |
| `desktop/.../PetView.swift` | 绘制水滴 UI + 湿度数值，低湿润度闪烁效果 |

## 八、玩法完整流程示例

**场景 1：正常使用**
> 早上开机，小龙虾满水 💧💧💧💧。上午天气晴朗，真实湿度 55%，每 20 分钟掉 1 格。中午变成 💧💧○○，小龙虾走路变慢了，偶尔擦汗。用户按 H 喷水，水花四溅，小龙虾开心跳起来，恢复到 💧💧💧○。

**场景 2：下雨天**
> 下午开始下雨，湿润度暂停衰减，每 15 分钟自动 +1 格。小龙虾打着伞，湿润度自动维持在满值。用户完全不用操心。

**场景 3：忘记照顾**
> 用户出门没管 Cardputer，3 小时后回来。晴天 + 湿度 35%，衰减快。小龙虾湿润度 0 格，趴在地上头顶冒 💧。天气配饰消失了，换成了小水壶。按空格也跳不起来。用户按 H 喷水 → 小龙虾站起来 → 再按几次 H → 恢复正常 → 天气配饰回来了。

**场景 4：桌面宠物同步**
> 用户在 Mac 上看到桌面小龙虾突然趴下了、走不动了，知道该去 Cardputer 上按 H 喷水了。
